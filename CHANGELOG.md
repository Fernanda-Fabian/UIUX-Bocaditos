

## Versión 1 – Wireframe funcional inicial

La primera versión del diseño se centró en establecer la estructura funcional del sistema, definiendo los tres roles principales: Usuario, Donador y Administrador. Cada pantalla fue construida con base en los requerimientos funcionales, sin aplicar estilo visual ni jerarquía gráfica.

- **Inicio de sesión**: campos básicos (email, contraseña), opción “Remember me” y botón de acceso.
- **Usuario**: visualización de matrícula, grupo, carrera y cuatrimestre. Se incluyen campos para alergias alimentarias y comentarios.
- **Donador**: formulario para registrar donaciones, incluyendo cantidad, tipo de comida, fecha de vencimiento y datos de entrega.
- **Administrador**: filtros de búsqueda por fecha y estado, navegación entre secciones clave (Inicio, Historial, Donaciones).
- **Rastreo de donaciones**: tabla con cinco registros, mostrando ID, fecha, donador, alimento, cantidad y estado (presentado fuera de la tabla).

Esta versión permitió validar la lógica del sistema, el flujo entre pantallas y la funcionalidad mínima esperada. No se aplicaron colores, tipografías ni elementos gráficos, ya que el objetivo era representar la arquitectura de información y los componentes funcionales.

---

## Versión 2 – Wireframe reorganizado y preparado para estilo visual

La segunda versión conserva la estructura funcional definida en v1, pero introduce mejoras en la organización visual, jerarquía de contenido y preparación para el prototipo de media fidelidad.

- **Inicio de sesión**: se mantiene la funcionalidad, pero se reorganizan los elementos para mejorar la legibilidad.
- **Usuario**: se conserva la información personal, pero se mejora la segmentación de campos y la claridad en la presentación.
- **Donador**: se reorganiza el formulario de donación, facilitando la lectura y anticipando el uso de componentes visuales.
- **Administrador**: se agrega una nueva sección de estadísticas por donador, mostrando porcentajes de participación (Walmart 35%, Chedraui 65%), lo que enriquece el análisis de impacto.
- **Rastreo de donaciones**: se mantiene la tabla con los cinco registros, pero ahora el estado de cada donación se integra directamente en la tabla, mejorando la interpretación visual.

Estas modificaciones responden a la necesidad de preparar el diseño para la aplicación de estilo visual, mejorar la experiencia de usuario y reforzar la coherencia entre pantallas. La evolución entre v1 y v2 no altera la lógica funcional, sino que la refuerza mediante una mejor distribución, jerarquía y claridad.

## Registro de versiones del diseño – Mockups v1, v2 y v3

La evolución del diseño visual del proyecto *Bocaditos* se desarrolló en tres etapas de mockup, cada una con objetivos distintos y niveles progresivos de fidelidad. A continuación se describe qué contenía cada versión, qué se modificó y por qué se tomaron esas decisiones.

---

### Mockup v1 – Propuesta inicial simplificada

La primera versión del mockup fue una aproximación visual básica, centrada únicamente en las pantallas de inicio de sesión y registro. Se diseñó con el objetivo de validar la estética general del sistema y explorar el uso de ilustraciones relacionadas con alimentos para generar empatía y conexión con el usuario.

- Se incluyeron campos para usuario, correo y contraseña.
- Se diseñaron botones de acceso y creación de cuenta.
- Se utilizaron ilustraciones de comida (sándwich y recipiente) como elementos centrales.
- No se integraron funcionalidades específicas por rol ni pantallas internas del sistema.

Esta versión fue útil para establecer una identidad visual inicial, pero resultó limitada en términos de funcionalidad y profundidad. Por ello, se decidió avanzar hacia una representación más completa del sistema.

---

### Mockup v2 – Expansión parcial del sistema

La segunda versión del mockup buscó ampliar el alcance del diseño, incorporando algunas pantallas internas del sistema y simulando interacciones básicas. Aunque se mantuvo una estética sencilla, se comenzó a representar la estructura funcional del wireframe v1.

- Se incluyeron pantallas para el panel del usuario y del administrador.
- Se representaron formularios de donación y campos de rastreo.
- Se mantuvo una estética limpia, sin aplicar aún la paleta de colores ni tipografías definitivas.
- Algunas secciones del wireframe v2 fueron omitidas o simplificadas.

Esta versión permitió validar la navegación entre pantallas y la coherencia entre roles, pero aún no reflejaba completamente la estructura funcional definida en el wireframe v2.

---

### Mockup v3 – Recreación completa del wireframe v2

La tercera versión del mockup representa una recreación fiel y completa del wireframe v2, integrando todas las pantallas funcionales y aplicando el estilo visual definido en la justificación del Sprint 3.

- Se incluyeron todos los roles: Usuario, Donador y Administrador.
- Se representaron formularios completos, filtros, tablas y estadísticas.
- Se aplicó la paleta de colores por rol, tipografías jerarquizadas y componentes visuales coherentes.
- Se respetó la estructura funcional del wireframe v2, incluyendo campos, navegación y distribución.

---

### Mockup v4 – Ajuste de diseño

En esta 4ta version del mockup se reajusto el diseño haciendolo mas manejable y teniendo una mejor usabilidad, se añadio un mejor menu en donde es mas claro y alineado con los diagramas antes vistos
- Se agrego un apartado de chat.
- Se ajustaron los menus.
- Se agregaron mas imagenes.


## Evolución del diseño

El diseño avanza desde una representación funcional mínima hacia una versión más estructurada y visualmente preparada. A medida que se consolidó la lógica del sistema, se optimizó la navegación, se definieron los roles con mayor precisión y se integraron componentes que anticipan el prototipo de media fidelidad. Este proceso iterativo, validado por criterios académicos, buscó mejorar la accesibilidad, la claridad y el impacto social del sistema.

En paralelo, los mockups permitieron trasladar esa estructura a una representación visual progresiva. La primera versión se planteó como una propuesta simplificada, centrada en las pantallas de acceso, con el objetivo de explorar la identidad gráfica y establecer una base estética. Posteriormente, se amplió el alcance visual incorporando algunas pantallas internas, aunque sin representar aún la totalidad del sistema.

Finalmente, se optó por recrear completamente la versión funcional más robusta en el mockup final. Esta última iteración integra todas las pantallas, formularios y componentes visuales, respetando la jerarquía informativa y aplicando el estilo definido. Con ello, el diseño evoluciona hacia una solución integral que combina funcionalidad, estética y propósito social, lista para validación académica.
